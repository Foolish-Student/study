# 1.5 Agent 的行动力：语言输出能力和工具使用能力（重点梳理）

## 1️⃣ 行动力在 Agent 里的位置

* 前面讲的是：**大脑（推理）+ 感知（看、听、理解世界）**
* 本节重点：**有了“想”和“看听”，还必须“能做”**，否则 Agent 只是“聪明顾问”，不是“能干活的助手”。
* 行动力主要来自两块：**语言输出能力 + 工具使用能力**。

---

## 2️⃣ 语言输出能力——用“说 / 写”完成一大半工作

**作用：**

* 面向人：写邮件、写报告、生成方案、解释结论、给建议 → 直接影响人的决策与协作。
* 面向系统/其他 Agent：用结构化文本（JSON、指令格式等）给出下一步要执行的动作。

**关键点：**

* 不只是“回答问题”，而是：

  * 能按**指定格式**输出（表格、步骤说明、Markdown 等）。
  * 能**解释自己的推理过程和理由**，让人类看得懂。
  * 能根据对象和场景调整语气、风格（正式/口语/总结/提示等）。

---

## 3️⃣ 工具使用能力——给 Agent 装上“手脚”

**什么是工具：**

* 各种可调用的外部能力：数据库/API、搜索引擎、内部业务系统、代码执行环境、文件读写、甚至物理设备接口等。

**典型闭环：**

1. Agent 根据当前目标和信息，判断**是否需要调用工具**。
2. 生成工具调用（选哪个工具 + 什么参数）。
3. 工具执行并返回结果（数据/状态/错误）。
4. 大模型再基于结果继续推理：是否完成？要不要再调用别的工具？要不要反馈给用户？

**关键点：**

* 让 Agent 拿到**最新、特定、私有**的数据，而不是只靠训练时记住的东西。
* 让它能**真正改变世界状态**（改订单、建任务、写文件等），而不仅仅是“嘴上说说”。
* 把大模型从“聊天模型”升级为“**任务执行中枢**”。

---

## 4️⃣ 两者合在一起：真正的“能干活的 Agent”

* **语言输出**：对人类/系统给出清晰可用的结果和指令。
* **工具使用**：对现实/业务系统做出实际操作。

合在一起，就形成：

> **感知（1.4） + 推理（1.3） + 行动（1.5）** 的完整闭环，
> 让 Agent 从“会聊天的模型”变成“能理解目标、能自己查 + 能自己干活的智能体”。
