# 2.5 Agent 的推理引擎：ReAct 框架（重点梳理）

## 1️⃣ ReAct 是什么？

* ReAct（Reasoning + Acting）是 **给大模型用的“推理 + 行动”认知框架**，由 Yao 等人在 ICLR 2023 论文中提出。
* 在众多框架（CoT、ToT、LLM+P 等）中，ReAct 被 LangChain、LlamaIndex 等主流工具采纳为 **默认推理引擎**，用来开发既能复杂推理、又能有效行动的 Agent。

👉 一句话：**ReAct = 把“大模型怎么想、怎么用工具去做事”串成一套标准套路。**

---

## 2️⃣ ReAct 的核心思想

* 目标：在**动态、不确定环境**中，让 Agent 能持续学习、适应，并把推理快速转成行动。
* 核心哲学：通过一个不断循环的 **“观察—思考—行动—再观察”** 闭环来做决策。

这个闭环是 ReAct 的本体，而不是一棵固定的决策树或写死的算法。

---

## 3️⃣ 三个关键步骤：Thought / Action / Observation

ReAct 把每一轮决策拆成 3 步：

1. **Thought（思考）**

   * 大模型基于当前信息进行推理：

     * 现在的情况是什么？
     * 有哪些可能的下一步？
     * 需要不要调用工具 / 搜索更多信息？

2. **Action（行动）**

   * 根据 Thought 的结果，选择并执行一个具体行动：

     * 调用某个工具（搜索、数据库、计算器等）
     * 或者给出一个中间回答 / 决策

3. **Observation（观察）**

   * 观察行动结果（工具的返回值、环境反馈）；
   * 把这些结果再次作为输入，进入下一轮 Thought。

→ 这样一轮轮循环，直到 Agent 认为任务已经完成，给出 Final Answer。

---

## 4️⃣ ReAct 相比“只推理 / 只行动”的优势

书中用 ReAct 对比了几种策略的效果：

* **只推理（Think only）**

  * 在脑子里瞎猜，不查外部信息，容易受模型内部知识局限，推着推着就错了。
* **只行动（Act only）**

  * 一直乱搜、乱查，但缺乏系统推理，不能好好整合信息，也容易得不出正确结论。
* **ReAct（Reason + Act）**

  * 先推理“该搜什么 / 该用什么工具”，再行动，再根据结果继续推理。
  * 每一步推理都有外部信息支撑，路径清晰、可追踪，更容易得到**准确且可解释的结果**，且不那么受模型内部记忆的限制。

👉 用人话讲：

> **ReAct = 一边动脑子想，一边出去打听消息、做实验，再回来接着想。**

---

## 5️⃣ ReAct 在 Agent 里的地位

* ReAct 为 Agent 提供了一个 **统一的“思考 + 用工具”循环模板**，非常适合连续决策、交互式任务（自动客服、问题求解、RAG 问答等）。
* 书中评价：

  * ReAct 是 Agent 设计的**里程碑式框架**，显著提升了 Agent 的推理与决策能力；
  * 随着技术与应用扩展，它及其衍生框架，很可能会成为智能 Agent 领域的主流范式。

---

你可以把 2.5 压缩成一句记忆：

> **ReAct = “观察—思考—行动—再观察”的循环框架，
> 让大模型不再只是想想而已，而是边想边查、边做边改，
> 成为 Agent 推理与调用工具的标准引擎。**
